# الأمان — التوثيق الكامل

## طبقات الحماية المنفّذة

---

### 1. نظام المصادقة

#### مشرف النظام
- كلمة مرور مشفّرة بـ **bcrypt** (`password_hash` + `password_verify`)
- **CSRF Token** على جميع نماذج POST — يمنع هجمات Cross-Site Request Forgery
- تجديد معرّف الجلسة عند كل تسجيل دخول (`session_regenerate_id(true)`)
- الجلسة تنتهي بإغلاق المتصفح (`session.cookie_lifetime = 0`)
- `session.cookie_httponly = 1` ← JavaScript لا يستطيع قراءة الكوكي
- `session.cookie_samesite = Strict` ← حماية إضافية من CSRF

#### الموظف
- لا يوجد كلمة مرور — الهوية عبر **unique_token** (64 حرف عشوائي)
- التوكن يُنتج بـ `bin2hex(random_bytes(32))` ← آمن كريبتوغرافياً
- لا يمكن تخمينه: احتمال 1 من 2^256

### 2. قفل الجهاز (Device Fingerprinting)

#### آلية العمل (الربط يدوي فقط — v3.2.0)
```
الدخول الأول (بدون ربط مفعّل):
  JS يحسب بصمة SHA-256 ← ترسل للـ API
  ← لا توجد بصمة محفوظة + وضع الربط غير مفعّل
  ← الموظف يدخل بدون ربط ✅

تفعيل الربط من المشرف:
  المشرف يضغط "ربط الجهاز" ← device_bind_mode = 1
  ← الموظف يفتح الرابط ← الجهاز يُربط تلقائياً ✅

الدخول التالي على نفس الجهاز (مربوط):
  JS يحسب نفس البصمة ← hash_equals() ← ✅

دخول من جهاز مختلف (مربوط):
  JS يحسب بصمة مختلفة ← hash_equals() يفشل ← ❌ رسالة قفل
```

> **ملاحظة:** لا يحدث ربط تلقائي. المشرف يتحكم بالكامل متى يتم ربط جهاز الموظف.

#### مكونات البصمة
1. `userAgent` — نظام التشغيل والمتصفح
2. `screen.width × height × colorDepth` — مواصفات الشاشة
3. `Intl timezone` — المنطقة الزمنية
4. `navigator.language` — لغة الجهاز
5. `canvas fingerprint` — بصمة رسومية فريدة للـ GPU

#### مقاومة التزوير
- `hash_equals()` في PHP ← يمنع **Timing Attacks**
- البصمة SHA-256 لا تُعكس ← المهاجم لا يستطيع استخراج البيانات الأصلية

### 3. الحماية من الموقع المزيّف

#### صيغة Haversine
حساب المسافة الفعلية على سطح الكرة الأرضية — يصعب التلاعب فيه لأنه يُحسب على الخادم:

```php
function isWithinGeofence(float $empLat, float $empLon): array {
    // الخادم يحسب المسافة — لا يثق بالحسابات القادمة من الجهاز
    $distance = calculateDistance($empLat, $empLon, $workLat, $workLon);
    return ['allowed' => $distance <= $radius, 'distance' => $distance];
}
```

> الحسابات الجغرافية تتم على **الخادم**، ليس على المتصفح.

### 4. الحماية من الحقن (Injection)

#### SQL Injection
- جميع استعلامات DB تستخدم **Prepared Statements + PDO**
- `PDO::ATTR_EMULATE_PREPARES = false` ← Prepared Statements حقيقية

```php
// ✅ آمن دائماً
$stmt = db()->prepare("SELECT * FROM employees WHERE unique_token = ?");
$stmt->execute([$token]);
```

#### XSS (Cross-Site Scripting)
- دالة `sanitize()` على جميع المدخلات: `htmlspecialchars + strip_tags + trim`
- `htmlspecialchars()` على كل إخراج ديناميكي في HTML

### 5. حماية الـ API

- `Access-Control-Allow-Origin: [SITE_URL]` ← CORS محدود لنطاق الموقع فقط
- التحقق من `$_SERVER['REQUEST_METHOD'] === 'POST'`
- قراءة البيانات من `php://input` + `json_decode` (لا `$_POST` — أكثر أماناً مع JSON)
- التحقق المتسلسل: Token ← Geofence ← وقت الدوام ← تكرار

### 6. حماية الملفات الحساسة

```apache
# .htaccess
<Files "install.php">
    Order Allow,Deny
    Deny from all
</Files>
```

- `install.php` محجوب بعد التثبيت
- مجلد `includes/` محمي من الوصول المباشر
- مجلد `docs/` ينبغي حجبه في بيئة الإنتاج
